## Summary

**Dental PACS in a Box.** Orthanc + OHIF as Docker stack, one command.

- **Differentiator:** Dental-First Open Source PACS (unclaimed category)
- **Use case 1:** Functioning tool for own dental practice (CBCT, OPG, Intraoral)
- **Use case 2:** Showcase for AI-assisted development (Conductor + Claude Code + gstack)
- **Use case 3:** Lead magnet / giveaway for Ambientwork customers
- **Use case 4:** Extensible to general medicine (CT, MRI)

## Chosen Approach

**Dental-Feature Sprint + Launch** (Approach B from /office-hours)

1. Sprint 0: OHIF Extension Spike (timeboxed 2-3 days)
2. Sprint 1: Dental Patient View (Hanging Protocols, terminology, study list)
3. Sprint 2: Dental Workflow Presets (Implant, Endo, Ortho)
4. Sprint 3: Launch Package (Landing Page, README, Screencast, Build-Log)

## EUREKA

Dental Imaging is NOT a subset of Medical Imaging. Dentists have fundamentally
different workflows, modalities (CBCT, OPG, Intraoral), and IT constraints.
"Medical PACS but simpler" is the wrong frame. "Dental-native stack" is its own category.
