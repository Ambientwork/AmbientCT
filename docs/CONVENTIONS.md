# Conventions — AmbientCT

## Branches
- `feat/<short-name>` — New features
- `fix/<short-name>` — Bug fixes
- `docs/<short-name>` — Documentation only
- `chore/<short-name>` — Config, CI, dependencies

## Commits
Conventional Commits: `feat: add drag-drop upload`, `fix: CORS header`, `docs: update README`

## Pull Requests
Include three sections:
1. **What changed** — 1-2 sentences
2. **How to test** — Steps to verify
3. **Screenshots** — If UI changed

## Code style
- Shell scripts: `shellcheck` clean, `--help` on every script
- JavaScript: Prettier defaults
- Docs: Written for non-technical practice administrators
